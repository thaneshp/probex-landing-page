How ProbeX works

ProbeX makes it easy to test your REST API for vulnerabilities and get a detailed analysis of its security status.

1. Enter your API URL

To start the testing process, simply enter the URL of your REST API into the ProbeX interface.

2. Analyze your API

Once you have entered your API URL and any additional headers or body, ProbeX will begin analyzing your API.

3. Review your results

Once the testing is complete, ProbeX will generate a report detailing any issues that were found.

Comprehensive Vulnerability Detection

ProbeX is designed to detect a wide range of vulnerabilities that could compromise the security of your APIs.

Data Leakage

ProbeX is designed to detect data leakage vulnerabilities that could expose sensitive information to unauthorized users.

Cross-site Scripting

Cross-site scripting (XSS) vulnerabilities, which could let an attacker insert malicious code or content into a website, can be found via ProbeX.

SQL Injection

SQL injection flaws that could let an attacker run arbitrary SQL commands on a database are detectable by ProbeX.

Broken Authentication

ProbeX is made to find flaws in authentication and session management procedures that could let an attacker access a system or resource without authorisation.

CSRF

Cross-site request forgery (CSRF) vulnerabilities can be found using ProbeX, which means an attacker won't be able to deceive a user into using a website or web application for nefarious purposes.

Rate Limit

ProbeX is made to identify rate limit flaws that could let an attacker get around rate-limiting measures and possibly go above the intended limit.

CORS Misconfiguration

Cross-origin resource sharing (CORS) rules that are improperly configured could be circumvented by an attacker, giving them access to or control over sensitive data or resources. ProbeX is able to identify these CORS controls.

JWT Attack

ProbeX is made to find JSON Web Token (JWT) flaws that could let an attacker take advantage of holes in JWT authentication's architecture to get unauthorised access to a system or resources.

CRLF Detection

Injecting malicious data into a website or online application could result in cross-site scripting (XSS) or other forms of attacks. ProbeX can identify CRLF vulnerabilities.
